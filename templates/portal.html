Firecrawl Guide

Firecrawl 集成使用步骤

Step 1 / 5

程序已经集成 Firecrawl

这里不是让你再装一个插件。本项目已经内置 Firecrawl 请求后端，可以调用 hosted Firecrawl 或你自托管的 Firecrawl API。

如果使用 hosted Firecrawl，API Key 绑定你的账号、额度和费用，所以程序不能也不应该替你生成。你只需要把自己的 Key 填到本页。

Step 2 / 5

怎么拿 API Key

点击下面按钮打开 Firecrawl 登录页，登录后点顶部 Dashboard。API Key 会出现在主屏，也可以进 API Keys 页面查看，复制以 fc- 开头的 Key。

如果你使用自托管 Firecrawl，把 API URL 改成你的自托管地址；如果自托管服务不需要 hosted key，可以按你的服务鉴权方式留空或填写对应 token。

打开 Firecrawl 登录页

Step 3 / 5

推荐最小配置

开启 Firecrawl，确认 API URL，填入 API Key。库存监控必须保持 maxAge=0，避免拿到缓存库存。

默认建议 storeInCache=false。zeroDataRetention 需要 Firecrawl 开通 ZDR；普通 hosted key 没开通时保持关闭。

Step 4 / 5

优先用于商品入库

Firecrawl 现在只作为外部兜底/诊断能力，不作为默认入库或定时监控路径。

商品入库优先使用 Scrapling 发现和抓取，再通过解析规则生成监控任务。

Step 5 / 5

Cloudflare 边界

本项目不内置 CAPTCHA、Turnstile 或 Cloudflare 绕过。Firecrawl 返回正常内容就解析；如果仍返回 challenge，会进入受保护站点冷却。

这种页面更稳定的兜底是 Webhook、手动录入，或者换一个公开可访问的商品页面。

导入商家会话

用于需要浏览器会话的商家页面。

1 用浏览器打开商品页，确认已经能看到真实商品卡片。
2 点击下方按钮复制脚本，在商品页控制台执行后复制输出。
3 粘贴 JSON 并保存，再点击检测确认库存状态。
控制台一键脚本
copy(JSON.stringify({domain:"",user_agent:navigator.userAgent,cookie_header:document.cookie},null,2))

仅在普通采集无法拿到真实商品 HTML 时使用。保存后会复用该域名会话进行轻量检测，到期后可重新导入。
